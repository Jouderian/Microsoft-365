# Nota de Pesquisa — Sessão 11/04/26

## Descobertas Técnicas

### Get-MgGroupMember e DirectoryObject

- `Get-MgGroupMember` retorna `DirectoryObject`, cujo **único campo nativo é `Id`**.
- Todos os campos adicionais (`displayName`, `userPrincipalName`, `mail`) chegam via `AdditionalProperties` quando solicitados com `-Property`.
- `@odata.type` **não pode ser incluído no `$select`** (causa HTTP 400), mas é **sempre retornado automaticamente** e disponível em `AdditionalProperties['@odata.type']`.
- Valores possíveis de `@odata.type`: `#microsoft.graph.user`, `#microsoft.graph.group`, `#microsoft.graph.device`, `#microsoft.graph.servicePrincipal`.

### bibliotecaDeFuncoes.ps1 — gravaLOG

- Parâmetro `[string]$tipo` sem valor vira `""` (string vazia), **nunca `$null`**.
- Chave `$null` em hashtable PowerShell é inválida — lança `InvalidNullKey`.
- O operador `-or` é **lógico booleano** em PowerShell — retorna `$true`/`$false`, jamais a string da direita.
- Solução correta para fallback de cor: `if (-not $color) { $color = 'White' }`.

## Decisões Tomadas

| Decisão | Justificativa |
|---------|---------------|
| UPN no lugar de `mail` para Grupos de Segurança | UPN é identificador canônico e sempre preenchido para usuários M365 |
| Mantido `PrimarySMTPAddress` para DLs | Exchange `ReducedRecipient` não expõe UPN; batch lookup foi descartado na sessão |
| `-Property` sem `@odata.type` no Get-MgGroupMember | API do Graph rejeita o campo no `$select`; ele é retornado automaticamente |
